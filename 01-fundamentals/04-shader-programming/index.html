Basic Shader Structure

                        
This example demonstrates the minimum required components of a WebGL shader program:

                        
    
                            
  • 
                                Vertex Shader: Transforms input vertices to clip-space positions using
                                gl_Position
                            
    • 
                            
  • 
                                Fragment Shader: Sets a constant color for all fragments using
                                gl_FragColor
                            
    • 
                            
  • 
                                Precision Qualifier: precision mediump float; sets the
                                floating-point precision
                            
    • 
                        

                        
Key concepts demonstrated:

                        
    
                            
  • The mandatory attribute qualifier for vertex input data
    • 
                            
  • The required gl_Position output in the vertex shader
    • 
                            
  • The required gl_FragColor output in the fragment shader
    • 
                        

                    


                    

                        
Varying Variables & Interpolation

                        
This example demonstrates how to pass data from the vertex shader to the fragment shader:

                        
    
                            
  • Attribute: Per-vertex color data is provided as an attribute
    • 
                            
  • 
                                Varying: The varying qualifier creates a variable that
                                passes from vertex to fragment shader
                            
    • 
                            
  • 
                                Interpolation: WebGL automatically interpolates varying values across
                                the face of primitives
                            
    • 
                        

                        
Key concepts demonstrated:

                        
    
                            
  • Declaration of matching varying variables in both shader stages
    • 
                            
  • Automatic perspective-correct interpolation of varying values
    • 
                            
  • Using multiple attributes (position and color) per vertex
    • 
                        

                    


                    

                        
Uniform Parameters

                        
This example demonstrates how to pass global values from JavaScript to shaders:

                        
    
                            
  • 
                                Uniform Variables: Global values that remain constant for all vertices
                                in a draw call
                            
    • 
                            
  • 
                                Matrix Transformation: Using a uniform matrix to transform vertices
                            
    • 
                            
  • Animation: Using time uniform to create dynamic visual effects
    • 
                        

                        
Key concepts demonstrated:

                        
    
                            
  • The uniform qualifier for global shader parameters
    • 
                            
  • Setting uniform values from JavaScript with gl.uniformXXX() functions
    • 
                            
  • Using uniform values to create time-based animations
    • 
                            
  • Matrix mathematics for 3D transformations
    • 
                        

                    


                    

                        
Simple Diffuse Lighting

                        
This example implements a basic diffuse lighting model:

                        
    
                            
  • Normal Vectors: Surface normals provided as vertex attributes
    • 
                            
  • View Space: Lighting calculations performed in view space
    • 
                            
  • 
                                Diffuse Reflection: Light intensity based on the dot product of normal
                                and light direction
                            
    • 
                        

                        
Key concepts demonstrated:

                        
    
                            
  • Transformation of normal vectors using the normal matrix
    • 
                            
  • Vector mathematics (normalization, dot product) for lighting
    • 
                            
  • Ambient and diffuse lighting components
    • 
                            
  • Light position as a uniform parameter
    • 
                        

                    


                    

                        
Texture Mapping

                        
This example demonstrates how to apply a texture to a 3D object:

                        
    
                            
  • 
                                Texture Coordinates: Per-vertex UV coordinates provided as attributes
                            
    • 
                            
  • 
                                Sampler: The sampler2D uniform type for accessing texture
                                data
                            
    • 
                            
  • 
                                Texture Lookup: Using texture2D() to sample colors from
                                the texture
                            
    • 
                        

                        
Key concepts demonstrated:

                        
    
                            
  • Passing texture coordinates from vertex to fragment shader
    • 
                            
  • Binding a WebGL texture to a sampler uniform
    • 
                            
  • Sampling texture data using interpolated coordinates
    • 
                            
  • Texture parameter configuration (filtering, wrapping)
