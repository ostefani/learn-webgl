WebGL 2 Data Flow: CPU-GPU Communication

Demonstrations of key data transfer mechanisms in WebGL 2

1. Basic Buffer Data Transfer with VAOs

A triangle with each vertex rendered in a different color

Technique Demonstrated:

This example shows the fundamental WebGL 2 data flow pattern:

  1. JavaScript arrays (positions, colors) created in CPU memory
  2. Vertex Array Object (VAO) created to store attribute state
  3. Buffer objects created and bound to appropriate targets
  4. Data transferred to GPU memory with bufferData()
  5. Attributes configured to read from buffers with vertexAttribPointer()

2. Interleaved Buffer Data

The same triangle but using an interleaved vertex data format

Technique Demonstrated:

This example demonstrates interleaved buffer organization with WebGL 2 VAOs:

  1. A single buffer containing both position and color data
  2. Data organized as [x,y,r,g,b, x,y,r,g,b, ...]
  3. Stride and offset parameters used to access elements
  4. VAO encapsulates all attribute bindings for efficient state management
  5. Improved memory locality and state change reduction

3. Uniform Data Transfer

A triangle with a rotating color effect controlled by uniforms

Technique Demonstrated:

This example shows uniform data transfer in WebGL 2:

  1. JavaScript calculates time-based color values
  2. Values transferred directly to shader uniform variables
  3. Uniform values affect all vertices in the same draw call
  4. No buffer required for per-draw-call data
  5. Uses WebGL 2 explicit fragment shader outputs
